# Try-To-Orbit

**Try-To-Orbit** is a C++ learning project that will gradually evolve into a small physics-based orbital simulation game. The long-term goal is to build a system that models gravitational mechanics and eventually supports a simplified moon-orbit style game, with future ideas including guidance logic inspired by Apollo-era systems.

The project begins as a command line physics console where users can experiment with basic orbital and gravity calculations. As more C++ concepts are learned, the program will expand step-by-step into a more complex simulation.

## Current Stage

The current version is an early **CLI mission physics console** that allows the user to:

- Run simple gravity and force calculations
- Input physical parameters such as mass or radius
- View computed results such as gravitational force or weight

This stage focuses on building strong foundations in:

- C++ syntax and program structure  
- Functions and decomposition  
- Constants and numerical calculations  
- Input and output handling  
- Clear program flow and logic  

The project will grow alongside progress through **LearnCpp** and other C++ resources.

## Project Philosophy

This repository is intended to document the process of learning and building systems-level C++ skills over time. Features will be added gradually as new language concepts are mastered.

Future expansions may include:

- orbital mechanics calculations
- simple simulation loops
- spacecraft state modeling
- physics-based gameplay
- eventual graphical or web integration

## Long-Term Goal

The long-term vision for **Try-To-Orbit** is a small physics-driven orbital game to Apollo 11 Assembly simulation environment written primarily in C++. The project is intended to demonstrate growth in:

- systems programming
- numerical modeling
- simulation architecture
